# 嵌套列表 Bug 修复 - 测试指南

## 修复内容

修复了 `createEnterKeymap()` 函数中的 bug，使其正确处理嵌套列表（非顶级 bullet）。

**主要改动：**
1. 检测当前行的缩进级别（`isNested`）
2. 修复 marker 正则支持缩进：`/^\s*([-*+]\s+)/`
3. 嵌套项仅保留缩进和 marker，不插入时间戳
4. 顶级项继续自动插入时间戳

## 编译状态

✅ **编译成功** - 无 TypeScript 错误

## 测试用例

### 用例 1：顶级列表自动插入时间戳（原有功能保持）

**操作步骤：**
1. 打开 Obsidian 笔记，找到 `## Journal` 区域
2. 输入：`- 06:42 完成报告`
3. 将光标放在行末
4. 按 `Enter`

**预期结果：**

## 部署到 Obsidian

### 方法 1：使用 deploy 脚本（推荐）

```bash
npm run deploy
```

这将：
1. 编译项目
2. 复制 `main.js`、`manifest.json`、`styles.css` 到本地 Obsidian 插件目录
3. 显示成功信息

### 方法 2：手动复制

```bash
npm run build
cp main.js manifest.json styles.css ~/.obsidian/plugins/journal-partner/
```

（注：上面的路径需要根据你实际的 Obsidian vault 位置调整）

## Obsidian 中重新加载

1. 打开 Obsidian
2. 按 `Cmd+P`（Mac）或 `Ctrl+P`（Windows）
3. 输入 `Reload app without saving`
4. 按 Enter

或者直接重启 Obsidian。

## 故障排查

### 问题：Obsidian 中新行仍然自动插入时间戳

**原因：** 插件没有重新加载

**解决：**
1. 重新运行 `npm run deploy`
2. 在 Obsidian 中重新加载插件（Cmd+P → Reload）

### 问题：新行没有 marker

**原因：** 可能行没有 marker，或者 marker 格式异常

**检查：**
- 确认行以 `- ` 或 `* ` 或 `+ ` 开头（有空格）
- 检查缩进是否为空格（不是 Tab）

### 问题：缩进不正确

**原因：** Obsidian 的缩进设置（Tab vs 空格）

**检查：** 在 Obsidian 设置中查看缩进配置
