# 全文搜索

## 目标与范围

在侧栏「搜索」Tab 提供对历史 Memo 条目的全文检索，以及一个「随机 memo」的惊喜入口，让用户翻旧账更省事。

**包含：**
- 搜索输入、结果列表、点击跳转
- 搜索索引的构建时机（懒加载）
- 随机 memo 视图

**不包含：**
- Obsidian 全库搜索（那是 Obsidian 自带能力，本插件不复刻）
- 按地点检索（见 [locations.md](locations.md)）

## 用户可感知行为

- 切到「搜索」Tab，第一次进入时会有短暂加载（建索引 / 扫描 daily notes）。
- 输入关键词，结果按相关性 / 时间排序（具体策略见「实现要点」）显示条目摘要与所属日期。
- 点击结果打开对应 daily note，并定位到该条目所在行。
- Tab 内提供「随机一条」按钮，从历史 Memo 里随机抽一条展示，用于回忆。

## 实现要点

- **索引来源**：扫描 daily notes 目录下所有 markdown 文件，对每篇文件用 [section.ts](../../src/section.ts) 的 `findSection` + `parseJournalEntries` 提取 Memo section 内的条目。
- **懒加载**：只在用户第一次切到搜索 Tab 时构建索引，避免拖慢插件启动；后续记录 Tab 有新增条目时增量更新缓存。
- **匹配策略**：按关键词做子串匹配（大小写不敏感），高亮命中片段。
- **跳转**：点击结果调用 Obsidian 的 `workspace.openLinkText` 打开文件，并把光标滚到目标行。
- **随机 memo**：从索引里等概率抽取一条，展示条目全文与日期，附一个「再来一条」按钮。

## 已知约束与遗留

- **搜索范围**：只搜 Memo section 内容，不搜 daily note 里其它区域；如果用户在 heading 之外写了内容，搜不到（这是刻意的，保持模块聚焦）。
- **索引在内存里**：不落地缓存，Obsidian 重启后首次进搜索 Tab 需要重建；对上千篇 daily note 的库来说尚可接受。
- **无高级语法**：暂不支持 `AND` / `OR` / 正则 / 按日期过滤等；后续可加。
- **无拼音搜索**：中文按字面匹配，输入拼音搜不到中文条目。
