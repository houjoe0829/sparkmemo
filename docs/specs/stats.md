# 统计

## 目标与范围

在侧栏「统计」Tab 展示年度热力图和汇总数字（字数、条目数、连续记录天数），让用户直观看到自己的记录节奏。计算逻辑放在纯函数模块里，方便单测。

**包含：**
- 分年度的每日条目数热力图
- 分年度 + 全部时间的汇总数字（字数、条目数、最长连续、当前连续）
- 「统计」Tab 的渲染

**不包含：**
- Memo section 之外的数据统计
- 复杂的图表（趋势线等，暂无）

## 用户可感知行为

- 切到「统计」Tab，看到：
  - 一个热力图（类似 GitHub contribution graph），格子颜色深浅代表当日条目数
  - 顶部若干汇总数字：本年 / 全部时间的条目数、字数、最长连续天数、当前连续天数
- 支持切换年份查看历史年度热力图。
- 数据来源是所有 daily notes 的 Memo section，切换 Obsidian 语言时数字格式跟随本地化（见 [i18n.ts](../../src/i18n.ts) 的 `currentLocale()`）。

## 实现要点

- **纯计算层**：[stats.ts](../../src/stats.ts) 提供 `computeStats(entriesByDate)`（或类似签名），输入 `{ 'YYYY-MM-DD': entries[] }`，输出汇总结构。刻意不依赖 Obsidian API，方便单测。
- **数据获取**：capture-view 的统计 Tab 复用搜索模块的索引扫描 daily notes → 按日期收集条目 → 传给 [stats.ts](../../src/stats.ts)。
- **字数口径**：字数按 section 里条目正文的字符（或分词）长度累加，具体规则见 [stats.ts](../../src/stats.ts) 实现。
- **连续天数**：以「当天存在至少一条 Memo 条目」为准计算 streak。
- **热力图渲染**：capture-view 里直接用 DOM + CSS 绘制方格；颜色深浅来自 CSS 变量，跟随主题。

## 已知约束与遗留

- **不依赖时间戳格式**：只看条目是否存在，不解析 `HH:MM`，所以自定义了 `timestampPattern` 也不影响统计。
- **无导出**：不支持导出年度报告 / 图片；后续可加。
- **性能**：遍历所有 daily notes，库很大时首次进入 Tab 可能略慢；靠搜索模块的懒加载缓存分担。
- **年份切换**：只按公历年份分组，不支持自定义起始月份（如财年、生日年）。
