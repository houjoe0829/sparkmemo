# 地点聚合

## 目标与范围

把所有带 `[名称](geo:lat,lon)` 标签的条目按城市聚合，让用户能翻看「我在某个城市写过哪些 memo」。城市名通过 OpenStreetMap Nominatim 逆地理编码得到。

**包含：**
- `geo:` 标签的识别与提取
- 逆地理编码调用与缓存
- 侧栏「地点」Tab 的城市分组渲染

**不包含：**
- 图片 EXIF 里的 GPS 提取本身（见 [capture-image.md](capture-image.md)）
- 地图可视化（暂无）

## 用户可感知行为

- 切到「地点」Tab，看到一个按城市分组的列表，每个城市下列出该城市写过的所有条目。
- 城市名首次出现时会有短暂 loading（在做逆地理编码），后续命中缓存直接显示。
- 网络不可用或坐标查无城市时，退化显示原始坐标或「未知位置」。
- 点击条目跳转到对应 daily note。

## 实现要点

- **标签识别**：正则匹配条目文本里的 `[label](geo:lat,lon)`，抽出 `lat` / `lon`。
- **逆地理编码**：[geocode.ts](../../src/geocode.ts) 调用 Nominatim 的 `reverse` 接口，返回 `city` / `country` 等字段；实现简单，无第三方依赖。
- **速率限制**：Nominatim 免费额度约 1 req/s；[geocode.ts](../../src/geocode.ts) 内做串行 + 间隔；结果按坐标网格 key 缓存到内存，避免同一位置反复查。
- **分组**：按 `city` 归组，无 city 的用 `country` 或「未知位置」兜底。
- **渲染**：capture-view 里的地点 Tab 按城市展开条目，点击复用搜索 Tab 的跳转逻辑。

## 已知约束与遗留

- **依赖公共 API**：Nominatim 是社区服务，网络阻塞或封 IP 时地点 Tab 会大面积显示坐标。可考虑接入其它服务或本地城市库作为兜底。
- **缓存不落地**：Obsidian 重启后缓存清空，重进 Tab 会重新查询；对经常访问的城市影响不大（走缓存），对稀有位置每次都要等。
- **精度粗**：只到城市级，不区分区县；对同城不同地点的旅行不友好。
- **无地图**：只有列表，没有把坐标画在地图上；后续可选。
- **占位文本**：图片附件写入时 label 是占位（如「位置」），是否回填成真实城市名由 UI 决定，不在此模块自动 rewrite 文件。
