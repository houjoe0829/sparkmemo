## Overview

**Journal Partner** is an Obsidian plugin that highlights and manages timestamps in journal entries. It automatically detects and renders `HH:MM` timestamps in designated journal sections as styled badges, with features for read-only protection, auto-insertion on Enter, and customizable styling.

## Build & Development Commands

```bash
# Install dependencies
npm install

# Development mode (watch and rebuild on save)
npm run dev

# Production build (minified, no sourcemaps)
npm run build

# Deploy to local Obsidian vault (requires configured VAULT_PATH in deploy.sh)
npm run deploy
```

The build pipeline:
- **TypeScript** → compiled to ES2018 via `tsc`
- **esbuild** → bundles into CommonJS `main.js`
- External dependencies (obsidian, @codemirror modules) are excluded from bundle
- Inline sourcemaps in dev, removed in production
- Output: `main.js` (plugin entry), `styles.css` (styles), `manifest.json` (metadata)

## Architecture

### Plugin Structure

**Main Components:**
1. **Editor Extensions** (`main.ts` lines 186–306) — Handles source & live-preview views via CodeMirror 6
2. **Reading View Post-Processor** (`main.ts` lines 310–363) — Handles rendered/preview views
3. **Settings Tab** (`main.ts` lines 413–566) — UI for user configuration
4. **Timestamp Detection** — Scoped section finder, line-based pattern matching

### Key Design Patterns

**Scoped Rendering:** The plugin only affects a user-specified heading section (default: `## Journal`). All rendering and interaction happens within `findSection()` bounds. This prevents unintended modifications elsewhere in the document.

**Dual-View Architecture:**
- **Editor View** (Source + Live-Preview): Uses CodeMirror 6 ViewPlugin with decorations + transaction filter for read-only enforcement
- **Reading View**: Post-processor walks the DOM and wraps timestamps in `<span class="jp-timestamp">`

Both views share the same timestamp detection logic but render differently.

**CSS-Driven Styling:** Colors are stored in CSS custom properties (`--jp-ts-color`, `--jp-ts-bg`) and applied via `applyCSSVariables()`. Settings changes dispatch `forceUpdateEffect` to trigger decoration recomputation without reloading.

### Critical Functions

- **`findSection(doc, headingName, headingLevel)`** — Locates the character range of a heading section by parsing the document line-by-line. Returns `null` if section not found. Used by timestamp detection, auto-timestamp insertion, and scoping logic.
- **`getTimestampRanges(doc, settings)`** — Returns an array of character ranges for each timestamp in the target section. Matches the pattern `^(?:[-*+]\s+)?(PATTERN)(?=\s|$)` to handle list markers.
- **`buildDecorations(doc, settings)`** — Creates a CM6 `DecorationSet` marking all timestamps. Called on construction and on updates.
- **`generateTimestamp()`** — Simple helper: returns current time as `HH:MM`.

### Enter Key Auto-Insertion

When Enter is pressed inside the journal section:
1. Cursor position is validated to be inside `findSection()` bounds
2. List marker (-, *, +) on the current line is detected
3. New line is inserted with the same marker + fresh timestamp + space
4. Cursor positioned after the timestamp
5. Implementation uses `Prec.high(keymap)` for high priority (overrides default Enter behavior)

### Read-Only Timestamps

The `readonlyFilter` transaction filter intercepts all document changes. If any change overlaps a timestamp range:
1. Transaction is rejected (returns `[]`)
2. User sees a toast notice: "⏰ 时间戳不可修改"
3. Detection uses range overlap: `fromA < to && toA > from`

### Checkbox Styling

When `circularCheckboxes` setting is enabled, the class `jp-circular-checkboxes` is added to `<html>`. CSS selectors with `!important` override checkbox `border-radius` for both editor and reading views:
- CodeMirror 6: `.cm-taskCheckbox`
- Obsidian task lists: `.task-list-item input[type="checkbox"]`

## Settings & Defaults

All settings are defined in `JournalPartnerSettings` interface with defaults in `DEFAULT_SETTINGS`:

| Setting | Type | Default | Purpose |
|---------|------|---------|---------|
| `targetHeading` | string | `"Journal"` | Heading text to activate plugin |
| `headingLevel` | number | `2` | Heading level (1–6, representing # to ######) |
| `timestampPattern` | string | `\d{2}:\d{2}` | Regex for timestamp detection |
| `timestampColor` | string | `#7c3aed` | Timestamp text color |
| `timestampBgColor` | string | `#ede9fe` | Timestamp background color |
| `readonlyTimestamps` | boolean | `true` | Block editing of existing timestamps |
| `autoTimestamp` | boolean | `true` | Auto-insert on Enter in journal section |
| `circularCheckboxes` | boolean | `false` | Render checkboxes as circles |

Settings are persisted via Obsidian's `this.saveData()` / `this.loadData()` API.

## Common Tasks

**Adding a new setting:**
1. Add field to `JournalPartnerSettings` interface
2. Add default to `DEFAULT_SETTINGS`
3. Add `Setting` control in `JournalPartnerSettingTab.display()`
4. Use `this.plugin.saveSettings()` in the onChange callback
5. Apply effect (e.g., call `refreshEditors()` if it affects decorations, or `applyCSSVariables()` if it's a style)

**Modifying timestamp detection:**
- Edit the regex pattern in `getTimestampRanges()` function or make it configurable via settings
- Test with various line formats: `- 06:42 ...`, `* 07:31 ...`, `+ 08:10 ...`, or non-list text starting with timestamp

**Changing styles:**
- Edit `styles.css` — no rebuild needed for CSS changes in development
- Use CSS custom properties for dynamic values that change via settings

**Testing locally:**
- Run `npm run deploy` to build and copy to Obsidian vault
- Reload plugin in Obsidian (Cmd+P → "Reload app without saving")
- Changes are immediately visible; no restart needed

## Known Implementation Details

- **Line-based timestamp detection:** Pattern matching happens per-line, not globally. List markers are required on the current line for timestamp to be detected at line start.
- **Newline handling:** Character offsets account for `\n` characters when building ranges.
- **DOM mutation in reading view:** Uses `TreeWalker` to find all text nodes, then replaces them. This preserves existing DOM structure (e.g., links, bold text within the paragraph).
- **ESBuild externals:** CodeMirror and Obsidian are excluded from the bundle — they're provided by Obsidian at runtime.
- **Dark theme:** Checkbox and timestamp styling automatically adapts via `.theme-dark` class and Obsidian CSS variables (e.g., `--background-secondary`, `--text-muted`).
