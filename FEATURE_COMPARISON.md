# 功能对比：修复前后 & 新增功能

## 📊 功能演示

### 场景 1：Enter 键 - 嵌套列表（修复）

#### ❌ 修复前（有 Bug）

```
输入：
## Journal
- 06:42 顶级任务
  - 07:31 子任务    ← 光标在行末按 Enter

输出（错误）：
## Journal
- 06:42 顶级任务
  - 07:3107:31      ← ❌ 时间戳被覆盖！bullet 结构破坏！
```

**问题：**
- ❌ 时间戳被错误地添加到嵌套项
- ❌ 前面的文本被覆盖
- ❌ bullet 结构破坏

#### ✅ 修复后（正确）

```
输入：
## Journal
- 06:42 顶级任务
  - 07:31 子任务    ← 光标在行末按 Enter

输出（正确）：
## Journal
- 06:42 顶级任务
  - 07:31 子任务
  -                ← ✅ 仅保留缩进和 marker，无时间戳
```

**修复点：**
- ✅ 嵌套项新行无时间戳
- ✅ 前面的内容完整保留
- ✅ bullet 结构正确

### 场景 3：Tab 键 - 顶级项缩进（新增功能）

#### ❌ 修复前（无此功能）

```
输入：
## Journal
- 06:42 任务        ← 光标在此行按 Tab

输出（修复前，无自动删除）：
## Journal
  - 06:42 任务      ← 缩进了，但时间戳仍在（不符合嵌套项规范）
```

**问题：**
- ❌ 缩进成为嵌套项后，时间戳还在
- ❌ 嵌套项不应该有时间戳（违反规则）

#### ✅ 修复后（新功能）

```
输入：
## Journal
- 06:42 任务        ← 光标在此行按 Tab

输出（新功能，自动删除时间戳）：
## Journal
  - 任务            ← ✅ 缩进 + 自动删除时间戳
```

**改进点：**
- ✅ 缩进后自动删除时间戳
- ✅ 符合嵌套项规范（嵌套项无时间戳）
- ✅ 结构一致性

## 🔄 完整工作流演示

### 创建日记列表的完整流程

#### 步骤 1-4：创建初始列表

```markdown
## Journal
- 06:42 早晨任务
- 08:30 会议
- 14:00 下午任务

每一行都是按 Enter 后自动添加时间戳的
```

#### 步骤 5：添加子任务（使用 Tab）

```
| 原来：
| - 06:42 早晨任务
| - 08:30 会议
| - 14:00 下午任务

↓ 点击 "会议" 行，按 Tab

| 结果：
| - 06:42 早晨任务
|   - 会议          ← 缩进，时间戳被删除 ✅
| - 14:00 下午任务
```

#### 步骤 6：在子任务下继续添加（按 Enter）

```
| 原来：
| - 06:42 早晨任务
|   - 会议
| - 14:00 下午任务

↓ 光标在 "会议" 行末按 Enter

| 结果：
| - 06:42 早晨任务
|   - 会议
|   -              ← 新子任务行，无时间戳 ✅
| - 14:00 下午任务
```

#### 最终结果

```markdown
## Journal
- 06:42 早晨任务
  - 07:00 检查邮件
  - 07:30 准备材料
- 08:30 会议
  - 08:35 记录要点
  - 09:00 后续行动
- 14:00 下午任务
  - 14:30 整理笔记
```

## 🔍 技术对比

### 正则表达式演进

#### Enter 键 Marker 匹配

```javascript
// 修复前（不支持缩进）
/^([-*+]\s+)/

// 修复后（支持缩进）
/^\s*([-*+]\s+)/
     ↑
   新增：支持可选空格
```

#### Tab 键时间戳匹配

```javascript
// 新增（Tab 功能）
new RegExp(`^([-*+]\\s+)(${plugin.settings.timestampPattern})\\s+`)
           ↑                 ↑                              ↑
        marker          时间戳模式                    后续空格
```

### 事务处理演进

#### Enter 键（修复前后）

```javascript
// 修复前：直接插入，可能产生冲突
insertion = '\n' + listMarker + ts + ' ';

// 修复后：根据缩进选择不同处理
if (isNested) {
  insertion = '\n' + currentIndent + listMarker;
} else {
  insertion = '\n' + listMarker + ts + ' ';
}
```

#### Tab 键（新增功能）

```javascript
// 新增：使用多个 change 的原子事务
const changes = [
  { from: deleteStart, to: deleteEnd, insert: '' },    // 删除时间戳
  { from: cursor.from, to: cursor.to, insert: '\t' }   // 插入 Tab
];
```
